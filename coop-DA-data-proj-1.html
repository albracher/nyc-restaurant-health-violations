Data Cleaning

Ok, so what does drop_na() do in R? From direct documentation: drops rows where any column (in this case, the entire dataset/dataframe) contains a missing value. So, in this case, anything with NA.

wtf is NA?

NA is different from “empty”; NA is considered “separate” and “missing”, “empty” or ““ is a separate thing completely.

So first, we need to add row numbers; this is pre-processing for later.

# throwing a row number on the original data so we can track stuff later
data_with_id <- data |>
  mutate(original_row = row_number())

My prof from undergrad described pivot_longer as this: “For when some of the column names are not names of variables, but values of a variable”

# we want to find where "NA" values are
na_locations <- data_with_id |>
  # first, just grab the rows that have an NA somewhere in them
  filter(if_any(-original_row, is.na)) |>
  # make everything a character so the pivot doesn't give us errors
  mutate(across(-original_row, as.character)) |>
  # convert our table to a long/increase rows and 
  pivot_longer(
    cols = -original_row,
    names_to = "column_name",
    values_to = "value"
  ) |>
  # now, just keep the lines that are NA
  filter(is.na(value))

# actually clean the data here, with the correct order of operations
cleaned_data <- data |>
  # remove rows with duplicate CAMIS/Date/Code
  distinct(CAMIS, INSPECTION.DATE, VIOLATION.CODE, .keep_all = TRUE) |> 
  # make the date columns "actual dates" first
  mutate(
    INSPECTION.DATE = mdy(INSPECTION.DATE),
    GRADE.DATE = mdy(GRADE.DATE),
    RECORD.DATE = mdy(RECORD.DATE)) |>
  # now, get rid of rows with NA and empty strings
  # why? mdy() will automatically convert "" into an NA, then drop_NA() will remove it/drop the row
  drop_na() |>
  filter(!if_any(where(is.character), function(x) x == ""))
# use count() to find which combinations have a count (n) greater than 1
duplicate_summary <- cleaned_data |>
  count(CAMIS, INSPECTION.DATE, sort = TRUE) |>
  filter(n > 1)
